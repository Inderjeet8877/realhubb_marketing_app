1. System Overview & Architecture

What the platform does

Realhubb is a WhatsApp Business CRM that lets you send individual & bulk messages, manage conversations in a real-time inbox, track message delivery status (sent → delivered → read with blue ticks), and receive replies from customers — all in one dashboard.

3. Firebase Setup

3.1 Create Firebase Project

STEP 1

Create project

Go to console.firebase.google.com → Add project → Enter name (e.g. realhubb-marketing-app) → Disable Google Analytics (optional) → Create project.

STEP 2

Enable Firestore

In the left sidebar → Firestore Database → Create database → Choose Start in production mode → Select region (e.g. asia-south1 for India) → Done.

STEP 3

Get Web App Config

⚙️ Project Settings → General tab → scroll to Your apps → Click </> Web → Register app → copy the firebaseConfig object. You need these for NEXT_PUBLIC_FIREBASE_* env vars.

STEP 4

Get Admin SDK credentials (service account)

⚙️ Project Settings → Service accounts tab → Generate new private key → Download the JSON file. Extract client_email and private_key for Vercel env vars.

3.2 Firestore Security Rules

Go to Firestore → Rules tab and paste the following rules, then click Publish:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // WhatsApp — webhook writes without auth, dashboard reads freely
    match /whatsapp_conversations/{docId} {
      allow read, write: if true;
    }
    match /webhook_logs/{docId} {
      allow read, write: if true;
    }
    match /whatsapp_templates/{docId} {
      allow read, write: if true;
    }

    // Everything else requires login
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
ℹ️

Why write: if true for WhatsApp collections? The webhook runs as a serverless function with no user auth context. Firebase Admin SDK bypasses rules, but these rules are also needed so the client SDK (browser onSnapshot) can read messages without the user needing special permissions.

4. Meta / WhatsApp Cloud API Setup

4.1 Create Meta App

STEP 1

Create new Meta App

Go to developers.facebook.com/apps → Create App → Select "Connect with customers through WhatsApp" use case → Enter app name → Select or create a Business Portfolio → Create app.

STEP 2

Connect WhatsApp Business Account

In the app → WhatsApp → API Setup → Connect to your WhatsApp Business Account (WABA). Note the WhatsApp Business Account ID shown on this page — you'll need it as WHATSAPP_BUSINESS_ACCOUNT_ID_1.

STEP 3

Add production phone number

WhatsApp → Phone Numbers → Add a phone number → Complete verification (OTP) → Done. Note the Phone Number ID — you need it as WHATSAPP_PHONE_NUMBER_ID_1.

STEP 4

Generate Permanent Access Token

1. Go to Meta Business Manager → Settings → System Users → Add new system user.
2. Assign assets → select your app → Full Control.
3. Assign your WhatsApp Business Account → Full Control.
4. Click Generate Token → add permissions: whatsapp_business_messaging, whatsapp_business_management.
5. Copy and save the token — this is META_ACCESS_TOKEN_1.

⚠️

Never use temporary tokens in production. The temporary token from API Setup expires in hours. Always use the permanent System User token for Vercel.

4.2 Configure Webhook

STEP 5

Set webhook URL and token

WhatsApp → Configuration → Webhook section:
• Callback URL: https://your-app.vercel.app/api/whatsapp/webhook
• Verify Token: whatsapp_verify_token_123 (or any secret string — must match WHATSAPP_WEBHOOK_VERIFY_TOKEN env var)
Click Verify and Save.

STEP 6

Subscribe to webhook fields

Under Webhook Fields → click Manage → enable the messages toggle → Save. This tells Meta to send inbound message events to your webhook.

STEP 7 ⭐ CRITICAL

Subscribe WABA to the app webhook (one-time)

After deploying to Vercel, open this URL in your browser once:
https://your-app.vercel.app/api/whatsapp/setup-webhook

You should see "success": true. This call does POST /{WABA_ID}/subscribed_apps which links your production WABA to the app so real inbound messages trigger your webhook.

Without this step, only Meta's test button will work — real user messages will be silently dropped.

🚨

Common Mistake: The webhook "Test" button in Meta's dashboard directly pings your URL — it bypasses WhatsApp's real message routing. It can pass even when real messages fail. Always test with a real phone sending to your business number after completing Step 7.

5. Environment Variables

These variables must be added in Vercel → Settings → Environment Variables AND in your local .env.local file for development. The .env.local file is in .gitignore and is never pushed to GitHub.

Firebase (Client SDK — browser)

NEXT_PUBLIC_FIREBASE_API_KEY
From Firebase Console → Project Settings → General → Web App config
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN
Format: your-project-id.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID
Your Firebase project ID (e.g. realhubb-marketing-app)
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET
Format: your-project-id.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID
Numeric sender ID from Web App config
NEXT_PUBLIC_FIREBASE_APP_ID
Format: 1:123456789:web:abcdef...

Firebase Admin SDK (server-side — bypasses Firestore rules)

FIREBASE_ADMIN_CLIENT_EMAIL
From service account JSON → client_email field (e.g. firebase-adminsdk-xxx@project.iam.gserviceaccount.com)
FIREBASE_ADMIN_PRIVATE_KEY
From service account JSON → private_key field. Include the full value with -----BEGIN RSA PRIVATE KEY----- and -----END RSA PRIVATE KEY-----. Paste as-is in Vercel.

Meta / WhatsApp Cloud API

META_ACCESS_TOKEN_1
Permanent System User access token from Meta Business Manager → System Users → Generate Token
WHATSAPP_PHONE_NUMBER_ID_1
Phone Number ID from Meta → WhatsApp → API Setup (numeric, e.g. 1043186245551497)
WHATSAPP_BUSINESS_ACCOUNT_ID_1
WABA ID from Meta → WhatsApp → API Setup (numeric, e.g. 1757017775671902)
WHATSAPP_WEBHOOK_VERIFY_TOKEN
Any secret string you choose — must match what you enter in Meta's webhook verify token field (e.g. whatsapp_verify_token_123)
ℹ️

After adding all variables in Vercel, click Redeploy (Deployments tab → 3 dots → Redeploy) for the new values to take effect.

6. Vercel Deployment

STEP 1

Push code to GitHub

# In the project directory
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/realhubb_marketing_app.git
git branch -M main
git push -u origin main
STEP 2

Connect to Vercel

Go to vercel.com → Add New Project → Import your GitHub repository → Select Next.js framework (auto-detected) → Do NOT deploy yet — add env vars first.

STEP 3

Add all environment variables

In the import screen → Environment Variables section → add every variable from Section 5 of this guide → then click Deploy.

STEP 4

Verify deployment

After deployment, open the app URL. Then test the webhook server is alive:
https://your-app.vercel.app/api/whatsapp/webhook
Should return: {"status":"webhook_online","ts":"..."}

STEP 5

Future deployments (automatic)

Every git push origin main triggers an automatic Vercel deployment. No manual action needed after initial setup.

8. Troubleshooting Checklist

8.1 Inbound messages not appearing in inbox

  • Open Vercel logs — is there a POST to /api/whatsapp/webhook when customer sends?
  • If NO POST in logs → WABA not subscribed. Call /api/whatsapp/setup-webhook once.
  • If POST exists but 500 error → check FIREBASE_ADMIN_CLIENT_EMAIL and FIREBASE_ADMIN_PRIVATE_KEY in Vercel
  • If POST exists, 200 OK, but no message in inbox → Firestore rules issue. Update rules in Firebase Console (Section 3.2)
  • Verify the webhook URL in Meta matches your Vercel URL exactly
  • Verify the verify token in Meta matches WHATSAPP_WEBHOOK_VERIFY_TOKEN in Vercel
  • Make sure "messages" field is subscribed under Webhook Fields in Meta

8.2 Outbound messages failing

  • Check META_ACCESS_TOKEN_1 is set in Vercel and is a permanent token (not temporary)
  • Check WHATSAPP_PHONE_NUMBER_ID_1 matches the Phone Number ID in Meta → API Setup
  • If sending a template: verify the template is APPROVED in Meta → Templates
  • If sending text reply: the customer must have messaged first within the last 24 hours
  • Check Vercel logs for the exact error from the WhatsApp API

8.3 Webhook verification failing ("couldn't be validated")

  • Visit the webhook URL in browser — should return JSON, not an error or HTML page
  • The verify token in Meta must exactly match WHATSAPP_WEBHOOK_VERIFY_TOKEN in Vercel
  • Vercel must have finished deploying the latest code (check Deployments tab — status: Ready)
  • If the page shows an error, check Vercel build logs for TypeScript or import errors

8.4 Messages API returning 500

  • Error "FAILED_PRECONDITION: requires index" → Remove orderBy from query and sort in JS instead
  • Error "permission-denied" → Firestore rules not updated. Publish rules from Firebase Console
  • Error "7 PERMISSION_DENIED" → Firebase Admin credentials wrong or missing in Vercel
  • Error "app/no-app" → NEXT_PUBLIC_FIREBASE_PROJECT_ID is missing

8.5 Delivery ticks not updating (staying at ✓)

  • Meta only sends delivered/read webhooks for messages sent with a valid permanent token
  • The wamid (WhatsApp message ID) must be saved when sending — check Firestore for wamid field
  • The webhook must be subscribed to the messages field (which covers statuses too)
  • onSnapshot listener must be active — check browser console for Firestore permission errors
